\section{Formal Problem Specification}
\label{sec:ch3problem}
% -----------------------------------------------------------------------------

\subsection{Restating the Core Limitation of Existing Frameworks}

As elucidated in Chapter~2, existing knowledge graph-constrained generation frameworks, prominently GCR \citep{luo2024gcr} and DoG \citep{li2024dog}, instantiate their constraint oracles as deterministic functions evaluated over the knowledge graph $\mathcal{G}$ and the identified question entities $\mathcal{E}_q$, expressed formally as $f(\mathcal{G}, \mathcal{E}_q)$. Whether computed entirely a priori or expanded topologically during decoding, the mathematical consequence is that the permitted valid token set $\mathcal{W}_{\text{val}}(t)$ at any arbitrary decoding step $t$ resolves to:
\begin{equation}
  \mathcal{W}_{\text{val}}^{\text{GCR}}(t) = f(\mathcal{G},\, \mathcal{E}_q)
  \quad \forall\, t
  \label{eq:gcr_oracle}
\end{equation}
Crucially, this specification is entirely independent of two critical contextual vectors: the semantic intent embedded within the natural language question $q$, and the dynamically evolving reasoning trajectory captured by the partial generation sequence $y_{<t}$. Consequently, all structurally accessible knowledge graph paths manifest symmetrically within the constraint hypothesis space, irrespective of their logical utility or factual relevance to the target query. For complex, multi-hop queries executing over dense subgraphs in Freebase, the combinatorial upper bound on this admitted valid set evaluates strictly to $|\mathcal{E}_q| \times d^3 \approx 8{,}000$ distinct traversable paths (referenced from Equation~\ref{eq:pathgrowth} in Chapter~2). Of this mathematically profound subset, strictly one path maps to the verifiable gold answer.

Such architectural paradigms erroneously conflate two orthogonal computational requirements. \textit{Structural validity} demands that any token permitted by the constraint oracle corresponds deterministically to a topological edge within the underlying knowledge schema---a baseline criteria satisfied universally by modern approaches. Conversely, \textit{Contextual relevance} necessitates that the permitted parameter space accurately reflects the task-specific semantic vector dictating generation step $t$---a stringent requirement universally ignored by contemporary mechanisms. This chasm between rigid topological correctness and fluid semantic applicability encapsulates the formal definition of the \textit{permissiveness problem}.

\subsection{The Upgraded Oracle Specification}

The DCA-Trie architecture resolves the permissiveness problem by formalising an upgraded mathematical constraint oracle that unifies structural accessibility with semantic integration. This upgraded function dynamically absorbs both the input semantic tensor and the unfolding generation prefix:
\begin{equation}
  \mathcal{W}_{\text{val}}^{\text{DCA}}(t) = f(\mathcal{G},\, \mathcal{E}_q,\,
  q,\, y_{<t})
  \label{eq:dca_oracle}
\end{equation}
where $q$ encapsulates the natural language query structure and $y_{<t} = (y_1, \ldots, y_{t-1})$ signifies the precise token history resolved at step $t$. To assert methodological completeness, this upgraded constraint vector must simultaneously satisfy three absolute conditions, which precisely operationalise the central objectives detailed in Chapter~1:

\begin{enumerate}
  \item \textbf{Structural Faithfulness (Condition~F):} Every candidate token $v \in \mathcal{W}_{\text{val}}^{\text{DCA}}(t)$ at every generation step $t$ must map conclusively to a mathematically valid sequence prefix of an existing path $p \in \mathcal{G}$. This axiom ensures that DCA-Trie rigorously observes a 100\% structural faithfulness baseline comparable to GCR and DoG, definitively preempting the introduction of any ungrounded lexical hallucinations into the constrained valid set.

  \item \textbf{Semantic Relevance Reduction (Condition~R):} The computed Semantic Irrelevance Ratio (SIR) of $\mathcal{W}_{\text{val}}^{\text{DCA}}(t)$ must demonstrate a strict mathematical reduction compared to the equivalent metric over $\mathcal{W}_{\text{val}}^{\text{GCR}}(t)$ when averaged globally across statistical evaluation corpora. Formally, denoting SIR as the primary diagnostic specified in Section~\ref{sec:ch3sir}:
        \begin{equation}
          \overline{\text{SIR}}\!\left(\mathcal{W}^{\text{DCA}}\right) <
          \overline{\text{SIR}}\!\left(\mathcal{W}^{\text{GCR}}\right)
          \label{eq:sir_condition}
        \end{equation}

  \item \textbf{Recall Preservation (Condition~P):} Evaluated within the context of semantic pruning, the verifiable gold reference path must mathematically evade spurious elimination by the scoring filter. Operationally, the empirical false negative rate (FNR) on the constrained WebQSP validation locus must evaluate strictly below the 5\% threshold:
        \begin{equation}
          \text{FNR} = \frac{|\{q \in \mathcal{Q}_{\text{val}} :
          p^*_q \notin \mathcal{W}_{\text{val}}^{\text{DCA}}(t)\}|}{|\mathcal{Q}_{\text{val}}|}
          < 0.05
          \label{eq:fnr}
        \end{equation}
        where $p^*_q$ resolves to the gold truth mechanism for question $q$ and $\mathcal{Q}_{\text{val}}$ encapsulates the dedicated 100-question held-out validation segment.
\end{enumerate}

These three fundamental stipulations transcend abstract diagnostic desiderata; they exist as deterministic targets quantified exhaustively in the Chapter~4 empirical framework. Condition~F is structurally verified by the raw faithfulness output metric. Condition~R translates directly to empirical observation via the proposed Semantic Irrelevance Ratio (SIR). Condition~P evaluates quantitatively against precision thresholds. Operating collectively, they form the core mathematical scaffolding constraining the DCA-Trie implementation.

\section{The Semantic Irrelevance Ratio: Definition and Measurement}
\label{sec:ch3sir}
% -----------------------------------------------------------------------------

\subsection{Motivation}

Standard evaluation metrics deployed for knowledge graph-constrained decoding architectures, specifically Hits@1, F1, and structural faithfulness rate, uniformly assess downstream generation outcomes: whether the terminal entity matches the gold answer, and whether the lexical output is strictly graph-valid. However, none mathematically verify the inherent precision of the constraint mechanism itself. A decoding framework could hypothetically achieve flawless target metrics while actively propagating the permissiveness problem: the underlying large language model might arrive at the correct terminal node by successfully traversing and discriminating among thousands of structurally valid but contextually irrelevant paths relying strictly on parametric memorisation, rather than because the constraint mechanism actually narrowed the conceptual search horizon. To resolve this critical diagnostic gap, this research introduces the Semantic Irrelevance Ratio (SIR) as a deterministic mathematical quantification of constraint efficiency independent of generation fidelity.

\subsection{Formal Definition}

Let $\mathcal{P}(q, t)$ formally denote the complete set of knowledge graph paths structurally admitted by the active constraint oracle during decoding execution step $t$ for a designated query $q$. A specific topological path $p \in \mathcal{P}(q, t)$ is definitively designated as \textit{semantically irrelevant} if its vectorised cosine similarity relative to the concatenated question-context embedding evaluates below a rigidly established baseline threshold $\tau_{\text{ref}}$:
\begin{equation}
  \text{irrelevant}(p, q, y_{<t}) = \mathbf{1}\!\left[
    \cos\!\bigl(E(p),\; E(\text{concat}(q, y_{<t}))\bigr) < \tau_{\text{ref}}
    \right]
  \label{eq:irrelevant}
\end{equation}
where $E(\cdot)$ represents the selected high-density \texttt{all-MiniLM-L6-v2} encoder embedding projection, and the boundary condition $\tau_{\text{ref}} = 0.3$ serves as a non-tunable static reference threshold formulated to securely delineate off-topic structural traversals without conflating underlying relevance verification with DCA-Trie's structurally dynamic admission boundary parameter $\tau$.

Consequently, the Semantic Irrelevance Ratio for a singular evaluated question $q$ isolating step $t$ is strictly defined as:
\begin{equation}
  \text{SIR}(q, t) = \frac{\sum_{p \in \mathcal{P}(q,t)}
    \text{irrelevant}(p, q, y_{<t})}{|\mathcal{P}(q, t)|}
  \label{eq:sir_step}
\end{equation}

The aggregate diagnostic SIR quantifying an entire generational framework across an independent evaluation corpus $\mathcal{Q}$ is derived by averaging uniformly over the sum of all queries intersecting all valid generation intervals up to the deterministic maximum traversal depth $L$:
\begin{equation}
  \overline{\text{SIR}} = \frac{1}{|\mathcal{Q}|} \sum_{q \in \mathcal{Q}}
  \frac{1}{L_q} \sum_{t=1}^{L_q} \text{SIR}(q, t)
  \label{eq:sir_aggregate}
\end{equation}
where $L_q$ strictly dictates the cumulative hop dimension of the generated lexical reasoning chain regarding query $q$.

\subsection{Properties and Interpretation}

The aggregate ratio $\overline{\text{SIR}}$ mathematically inhabits the interval $[0, 1]$. A terminal limit converging on $\overline{\text{SIR}} = 1$ confirms structural catastrophe constraint-wise: every route permitted by the structural oracle inherently lacks textual relevance regarding the primary question, effectively forcing the parametric LLM layer to shoulder the absolute burden of logical traversal. By definitive contrast, an evaluated baseline of $\overline{\text{SIR}} = 0$ perfectly asserts that every generated constraint subset represents intense, contextual target relevance---affirming perfect spatial restriction. Extant frameworks including GCR and DoG systematically execute near the maximum boundary, mathematically confirming massive structural SIR overhead specifically inflating at three and four-hop traversal bounds, where exhaustive path enumeration explodes. The implementation of the DCA-Trie architecture mathematically executes an intentional, quantifiable suppression of the overall SIR calculation, aggressively bypassing semantically tangential routes leveraging the active threshold mechanisms mathematically specified via Equation~\eqref{eq:dca_oracle}. Following standardized practices, empirical SIR outputs will inherently evaluate stratified across varied discrete hop depths to determine if active semantic exclusion consistently cascades scaling parallel to mounting query complexity.

\section{Semantic Relevance Scoring Mechanism}
\label{sec:ch3scorer}

\subsection{Scorer Architecture}

The semantic relevance scoring module natively operationalises the foundational architectural paradigm shift from the static function $f(\mathcal{G}, \mathcal{E}_q)$ to the dynamic formulation $f(\mathcal{G}, \mathcal{E}_q, q, y_{<t})$. The mathematical operation dictates receiving an individual discrete structural candidate knowledge graph path $p$, the raw natural language query sequence $q$, and the continuous partial derivation $y_{<t}$, synthesising these components into an executable continuous scalar relevance output within the $[0, 1]$ interval space.

An individual topological search path $p = (e_0, r_1, e_1, \ldots, r_L, e_L)$ is sequentially serialised into an interpretable natural language sequence via string concatenation bounded by space delimiters isolating consecutive structural nodes and edges:
\begin{equation}
  \text{str}(p) = e_0 \;\|\; r_1 \;\|\; e_1 \;\|\; \cdots \;\|\; r_L \;\|\; e_L
  \label{eq:pathstr}
\end{equation}
where extracted entity descriptors mathematically map to resolved Freebase human-readable lexical names (decoded computationally from internal MIDs wherever semantically viable), and intermediate formal relation vectors denote Freebase relation schema schemas where internal programmatic underscores undergo regular expression space replacement. Converging symmetrically, the dynamic querying context synthesises string integration parsing both spatial origin and topological position separating by an explicit lexical breakpoint string:
\begin{equation}
  \text{query}(q, y_{<t}) = q \;\|\; \texttt{[SEP]} \;\|\;
  \text{str}(y_{<t})
  \label{eq:querystr}
\end{equation}
where $\text{str}(y_{<t})$ effectively materialises the locally detokenised historical chain progression.

Sequentially, both string parameters $\text{str}(p)$ and $\text{query}(q, y_{<t})$ map deterministically through the robustly dense \texttt{all-MiniLM-L6-v2} transformer encoder block \citep{reimers2019sentence} translating natural linguistics into unified 384-dimensional spatially dense representation coordinates. The terminal absolute relevance scalar quantifies their continuous spatial cosine geometry:
\begin{equation}
  \text{score}(p,\, q,\, y_{<t}) = \cos\!\bigl(
  E\!\left(\text{str}(p)\right),\;\;
  E\!\left(\text{query}(q, y_{<t})\right)
  \bigr)
  \label{eq:scorer}
\end{equation}

Thus, an iteratively assessed semantic candidate path $p$ earns mathematical admission to the local continuous valid lexicon $\mathcal{W}_{\text{val}}^{\text{DCA}}(t)$ explicitly if and only if its geometric divergence confirms $\text{score}(p, q, y_{<t}) \geq \tau$, where $\tau$ constitutes the active parameterised admission magnitude threshold. This decisive binary activation directly embeds the continuous multi-dimensional similarity computation deeply within the robust logit-masking schema originally modeled natively as Equation~\eqref{eq:logitmask} in Chapter~2. \subsection{Threshold Selection}

The absolute admission threshold magnitude $\tau$ is systematically isolated via a precise mathematical evaluation over a structurally detached validation partition housing strictly 100 uniquely formulated, multi-hop question vectors isolated carefully from the primary WebQSP general evaluation run. The cardinal selection methodology algorithm prioritizes absolute empirical preservation of Condition~P (false negative containment) iteratively while executing maximum mathematical decay targeting overall base trie proportions sequentially relative to GCR baseline distributions. Analytically, model $\tau$ tests across continuous limits bounded safely within $[0.1, 0.6]$, increasing deterministically by $0.05$ per epoch lockstep. The optimized scalar defines definitively as the highest continuous bounding $\tau$ empirically conserving $\text{FNR} < 0.05$. This isolation routine necessarily fires independently separated: initially standardizing explicitly for DCA-Trie v1 (where global path threshold limits execute synchronously upfront during initialization block) and recursively calibrating optimally regarding DCA-Trie v2 (wherein progressive filter pruning necessitates active node recalibration). Differential model environments naturally force asynchronous threshold calibrations since v2 actively computes expanding historical depth sequentially parsing progressively precise vectors supporting naturally stricter limits scaling without data attrition.

\subsection{Encoder Caching}

To mathematically negate heavy repetitive recursive evaluation computation overlapping deeply during exponential parallel beam searches, every topological node vector embedding structurally encapsulated by $E(\text{str}(p))$ specifically mapped to known static tree arrays is explicitly parsed and stored a priori preceding active step execution. This pre-computational bypass guarantees efficiency intrinsically feasible specific to DCA-Trie v1 where path nodes lock during strict initialization. Alternatively within DCA-Trie v2, sequential array vector generations parsing actively traversing unspooled branches execute rigorously lazily (strictly mapped during node instantiation) and strictly encode caching limits ensuring immediate parallel overlap checks efficiently mitigate unnecessary recursive encoder calls inherently. Conversely, the dynamic target query scalar $E(\text{query}(q, y_{<t}))$ is inevitably regenerated uniquely immediately pursuing every partial beam update triggering strict modification instances (e.g., following discrete committed contextual entity iterations systematically appending the logical trail block).

\subsection{Algorithm}

\begin{algorithm}[htbp]
  \caption{DCA-Trie v1 — Static Semantic Filtering at Construction Time}
  \label{alg:v1}
  \begin{algorithmic}[1]
    \Require Knowledge graph $\mathcal{G}$; question entities $\mathcal{E}_q$;
    question $q$; max hop depth $L$; admission threshold $\tau$;
    sentence encoder $E$
    \Ensure Semantically filtered KG-Trie $\mathcal{T}^{\text{v1}}$

    \State \textbf{Precompute query embedding:}
    $\mathbf{u} \gets E(\text{query}(q, \varepsilon))$
    \Comment{$y_{<t} = \varepsilon$ at construction time}

    \State \textbf{Enumerate paths:}
    $\mathcal{P} \gets \textsc{BFS}(\mathcal{G}, \mathcal{E}_q, L)$
    \Comment{All paths within $L$ hops, as in GCR}

    \State \textbf{Score and filter:}
    \For{each path $p \in \mathcal{P}$}
    \State $\mathbf{v}_p \gets E(\text{str}(p))$
    \Comment{Precompute and cache path embedding}
    \State $s_p \gets \cos(\mathbf{u}, \mathbf{v}_p)$
    \If{$s_p \geq \tau$}
    \State Add $p$ to $\mathcal{P}_{\text{filtered}}$
    \EndIf
    \EndFor

    \State \textbf{Construct filtered trie:}
    $\mathcal{T}^{\text{v1}} \gets \textsc{BuildTrie}(\mathcal{P}_{\text{filtered}})$

    \State \Return $\mathcal{T}^{\text{v1}}$

    \Statex
    \Statex \textbf{At generation time} (identical to GCR):
    \For{each step $t$}
    \State $\mathcal{V}_t \gets \textsc{TrieLookup}(\mathcal{T}^{\text{v1}}, y_{<t})$
    \State $\mathbf{m}_t[v] \gets \mathbf{1}[v \in \mathcal{V}_t]$ for all $v$
    \State $\tilde{\boldsymbol{\alpha}}_t \gets \mathbf{m}_t \odot
      \textsc{LLMLogits}(y_{<t}, x)$
    \State $y_t \gets \text{beam-search-step}(\tilde{\boldsymbol{\alpha}}_t)$
    \EndFor
  \end{algorithmic}
\end{algorithm}

\subsection{Complexity Analysis}

The initial vector-grounded offline compilation matrix cost of structuring $\mathcal{T}^{\text{v1}}$ deterministically evaluates to mathematical $O(|\mathcal{P}| \cdot d_E)$, formally bounded by the absolute enumeration magnitude $|\mathcal{P}|$ harvested by the preliminary BFS topological expansion and constrained directly by the dense dimensional projection $d_E = 384$. However, this calculation is amortised entirely as a solitary fixed-cost offline phase executing strictly prior to temporal token expansion limits. Following compilation, sequential runtime generative complexity sustains an optimal mathematical constant $O(d)$, identically mirroring baseline GCR performance. This holds theoretically since generation token lookup querying specifically traversing $\mathcal{T}^{\text{v1}}$ functionally executes exact identical $O(1)$ child branch hashing parallel to the unfiltered structural counterpart. Conversely, RAM constraint parameters regarding $\mathcal{T}^{\text{v1}}$ optimize tightly to $O(|\mathcal{P}_{\text{filtered}}| \cdot L)$, demonstrating a severe proportional scalar structural reduction compared against massive GCR array matrices uniformly bounding at $O(|\mathcal{P}| \cdot L)$, mathematically directly proportionate to the observed rejection filtration parameter.

\subsection{Faithfulness Guarantee}

Consequently, the DCA-Trie v1 execution logically asserts absolute structural faithfulness strictly by architectural definition. Every scalar path strictly verified mathematically within the retained set $\mathcal{P}_{\text{filtered}}$ organically inherits entirely from the strict generation boundary $\textsc{BFS}(\mathcal{G}, \mathcal{E}_q, L)$. By rigorous definition, this primary origin subset guarantees absolute structural coherence wherein every sequential local triplet $(e_{i-1}, r_i, e_i) \in \mathcal{T}$ verifies rigorously against deterministic graph edges. Executing dense multi-vector distance exclusions removes exclusively mathematically valid subsets falling short relative to semantic correlation vectors, rather than interpolating novel fabricated tokens into the search tree matrix. Consequently, every token array traversing through the modified structure $\mathcal{T}^{\text{v1}}$ continuously maps specifically to formally sound underlying valid prefixes, entirely confirming the rigorous mathematical constraints defined by Condition~F.

\subsection{Design Rationale}

DCA-Trie v2 formally operationalises the complete continuous specification delineated in Equation~\eqref{eq:dca_oracle} by aggressively conditioning the structural constraint oracle upon the fluidly evolving sequential dimension $y_{<t}$ at strictly every evaluation tick. The mathematical motivation asserts that $y_{<t}$ strictly conveys deterministic progression parameters invisible to the standalone initial target $q$: specifically, it sequentially verifies exactly which logical entity trajectories and relational mappings have been irreversibly committed within the active analytical chain, strictly orienting subsequent structural vectors directly in alignment with the underlying parametrised search heading. Illustratively, if the latent decoding matrix executes a positional commit regarding \textit{Elvis\_Presley $\to$ starred\_in $\to$ Blue\_Hawaii}, purely mathematically valid subsequent evaluations strictly bind exclusively to \textit{Blue\_Hawaii}. Consequently, the dynamic context vector $E(\text{query}(q, y_{<t}))$ mathematically materialises this orientation strictly by condensing original natural semantics congruently with the dynamically unfolding logical premise structure.

Architecturally, DCA-Trie v2 inherently adopts the foundational continuous traversal topology proposed natively within DoG \citep{li2024dog}. However, it critically overrides DoG's explicitly naive structurally blind traversal logic (formalised originally via Equation~\ref{eq:dog} within Chapter~2) substituting a highly supervised semantically bounded expansion paradigm. Following every formal entity commit cycle $e_t$, the valid constraint tree extends explicitly only mapping strictly verified structural bounding neighbours corresponding to $e_t$ evaluating identically at or beyond the spatial gating threshold $\tau$:
\begin{equation}
  \mathcal{T}_{t+1} = \mathcal{T}_t \;\cup\; \bigl\{(e_t,\, r,\, e') :
  (e_t, r, e') \in \mathcal{T}_{\mathcal{G}} \;\wedge\;
  \text{score}(e', q, y_{<t}) \geq \tau\bigr\}
  \label{eq:v2expansion}
\end{equation}
where $\mathcal{T}_{\mathcal{G}}$ denotes the absolute bound enclosing all graph triplets. This fundamental substitution---transforming unconstrained scalar concatenation into aggressively gated vector intersection---functions as the primary structural derivation demarcating strictly bounded DCA-Trie v2 mechanics against permissive DoG arrays. \subsection{Algorithm}

\begin{algorithm}[htbp]
  \caption{DCA-Trie v2 — Step-Wise Semantic Expansion}
  \label{alg:v2}
  \begin{algorithmic}[1]
    \Require Knowledge graph $\mathcal{G}$; question entities $\mathcal{E}_q$;
    question $q$; admission threshold $\tau$; sentence encoder $E$;
    LLM with vocabulary $\mathcal{V}$
    \Ensure Generated reasoning chain $y_1 \ldots y_T$

    \State \textbf{Initialise trie} with entity nodes for $\mathcal{E}_q$:
    $\mathcal{T}_0 \gets \{e_q : e_q \in \mathcal{E}_q\}$

    \State \textbf{Precompute query embedding:}
    $\mathbf{u}_0 \gets E(\text{query}(q, \varepsilon))$

    \For{each step $t = 1$ \textbf{to} $T$}
    \State $\mathcal{V}_t \gets \textsc{TrieLookup}(\mathcal{T}_{t-1}, y_{<t})$
    \Comment{Valid tokens from current trie}
    \State $\mathbf{m}_t[v] \gets \mathbf{1}[v \in \mathcal{V}_t]$ for all $v$
    \State $\tilde{\boldsymbol{\alpha}}_t \gets \mathbf{m}_t \odot
      \textsc{LLMLogits}(y_{<t}, x)$
    \State $y_t \gets \text{beam-search-step}(\tilde{\boldsymbol{\alpha}}_t)$

    \If{$y_t$ is an \textit{entity} token $e_t$}
    \Comment{Expansion triggered at entity commits}
    \State \textbf{Update query embedding:}
    $\mathbf{u}_t \gets E(\text{query}(q, y_{\leq t}))$
    \State \textbf{Expand trie:}
    \For{each $(e_t, r, e') \in \mathcal{T}_{\mathcal{G}}$}
    \State $\mathbf{v}_{e'} \gets E(\text{str}((e_t, r, e')))$
    \Comment{Cache if previously computed}
    \State $s \gets \cos(\mathbf{u}_t, \mathbf{v}_{e'})$
    \If{$s \geq \tau$}
    \State Add $(e_t, r, e')$ to $\mathcal{T}_t$
    \EndIf
    \EndFor
    \Else
    \State $\mathcal{T}_t \gets \mathcal{T}_{t-1}$
    \Comment{No expansion at relation token steps}
    \EndIf
    \EndFor
    \State \Return $y_1 \ldots y_T$
  \end{algorithmic}
\end{algorithm}
